## 1.3 LIMIT output rows
We can use LIMIT to restrict the output to just the first few rows from the query. 
never EVER run select * from some_big_sales_table without a limit because there was over 1 trillion rows of data.

```SELECT * FROM dvd_rentals.actor LIMIT 10;  ```

* In some SQL flavours like SQL Server and Teradata, TOP is used instead of LIMIT and goes in the front of the SELECT statement.

```
SELECT
  TOP 10 *
FROM some_schema.table_name;
```

* Additionally - Postgres also allows for an OFFSET value to specify the number of rows to skip before taking the records based off the LIMIT clause.

# 2. Sorting Query Results

We can use the ORDER BY clause at the end of our queries to sort the output.

By default, the sorting will be done in ascending order and we can perform multi-level sorting by specifying more than one column.

One more thing about NULL values also - by default PostgreSQL will put all null values last unless specified with a NULLS FIRST 

## 2.1. Sort By Text Column

SELECT country
FROM dvd_rentals.country
ORDER BY country
LIMIT 5;

# NULL FIRST

When we put the NULLS FIRST expression at the end of the ORDER BY clause we will see a different output:

```
WITH test_data (sample_values) AS (
VALUES
(null),
('0123'),
('_123'),
(' 123'),
('(abc'),
('  abc'),
('bca')
)
SELECT * FROM test_data
ORDER BY 1 NULLS FIRST;
```

## 2.2. Sort By Numeric/Date Column

Sorting for any column with numbers, dates, timestamps is done from lowest to highest or latest to earliest for date time related columns.

```
-- order by (col_num)
SELECT
  *
FROM dvd_rentals.sales_by_film_category
ORDER BY 2
LIMIT 5;
```

## 2.3 Sort By Descending
We can also use the keyword DESC after the ORDER BY to reverse the sort order.

This is useful for text fields but is most common when trying to find the largest numeric value of a column or the latest date of a specific date time column.

Select multiple columns and also specifically choose which column you want to use in the ORDER BY clause.
```
SELECT
  category,
  total_sales
FROM dvd_rentals.sales_by_film_category
ORDER BY total_sales
LIMIT 1;
```

```
SELECT
  payment_date
FROM dvd_rentals.payment
ORDER BY payment_date DESC
LIMIT 1;
```

## 2.4. Sort By Multiple Columns -- super SUPER SUPER important 
We can also perform a multi-level sort by specifying 2 or more columns with the ORDER BY clause.
so to make it really extra clear - I will use this following sample example dataset to explain further.

sample_table

id	column_a	column_b
1	0	A
2	0	B
3	1	C
4	1	D
5	2	D
6	3	D

this sample_table is already sorted by column_a and column_b in ascending order.

```
DROP TABLE IF EXISTS sample_table;
CREATE TEMP TABLE sample_table AS
WITH raw_data (id, column_a, column_b) AS (
 VALUES
 (1, 0, 'A'),
 (2, 0, 'B'),
 (3, 1, 'C'),
 (4, 1, 'D'),
 (5, 2, 'D'),
 (6, 3, 'D')
)
SELECT * FROM raw_data;
```

### 2.4.1. Both Ascending - default
```
SELECT * FROM sample_table
ORDER BY column_a, column_b
```

4	    1	        D
5	    2	        D

## 2.4.2. Ascending & Descending
* col A desc and when values are equal in col a desc col b
```
SELECT * FROM sample_table
ORDER BY column_a DESC, column_b
```

Or we can put the DESC after both columns

## 2.4.5. Summary
Although this example dataset is quite simple - I can’t stress just how important this concept of ordering by multiple columns is. It becomes very very important when we start tackling window functions and other analytical techniques.

As you might have noticed, the sorting really impacts the order of rows that appear once we run the query. This is the crux of many many operations that we will cover in future.

## 2.5  Example Sorting Questions

* Q - Which customer_id had the latest rental_date for inventory_id = 1 and 2?

# 3. Exercises

## 3.1 What is the name of the category with the highest category_id in the dvd_rentals.category table?
```

SELECT name, category_id
from dvd_rentals.category
order by category_id DESC
```
## 3.2 For the films with the longest length, what is the title of the “R” rated film with the lowest replacement_cost in dvd_rentals.film table?
```
SELECT title, replacement_cost, length, rating
FROM dvd_rentals.film
order by length DESC, replacement_cost
LIMIT 10;
```
## 3.2 Who was the manager of the store with the highest total_sales in the dvd_rentals.sales_by_store table?
```

SELECT manager, total_sales
FROM dvd_rentals.sales_by_store
order by total_sales DESC
```
## 3.3 What is the postal_code of the city with the 5th highest city_id in the dvd_rentals.address table?
```
SELECT postal_code
FROM dvd_rentals.address
ORDER BY city_id DESC
LIMIT 1 OFFSET 4;
-- OFFSET 4 - Skips the first 4 rows - Leaves the 5th row
```
